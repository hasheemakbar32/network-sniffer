# Basic Network Sniffer in Python

This project implements a basic network sniffer using Python. The sniffer captures network packets from the network interface and extracts important information such as source IP address, destination IP address, protocol, and TTL value.

The goal of this project is to understand how network packets flow through a network and how packet headers are structured.

---

## 1. Objective of the Project

The objective of this project is to build a basic network sniffer that can capture and analyze network packets. The sniffer reads raw packets from the network interface and extracts useful information from the IP header.

The program displays:

- Source IP Address
- Destination IP Address
- Protocol Number
- Time To Live (TTL)

---

## 2. Tools & Technologies

The following tools and technologies were used in this project:

- Python 3
- Socket Library
- Struct Library
- Linux / Kali Linux
- Raw Sockets

---

## 3. How Network Sniffing Works

A network sniffer captures packets directly from the network interface card (NIC). These packets contain multiple layers of information such as Ethernet headers, IP headers, transport layer headers, and payload data.

The packet structure looks like this:

Ethernet Header → IP Header → TCP/UDP Header → Payload

This project extracts the IP header to analyze the packet details.

---

## 4. Project Architecture

The working flow of the sniffer is:

1. Create a raw socket.
2. Capture network packets.
3. Extract Ethernet header.
4. Identify IPv4 packets.
5. Extract IP header information.
6. Display packet details.

---

## 5. Implementation

The program uses Python raw sockets to capture network packets. It reads packet data from the network interface and parses the Ethernet and IP headers using the struct module.

The sniffer continuously listens for incoming packets and prints packet information in the terminal.

## 7. Testing the Sniffer

To test the sniffer:

Run the program using root privileges:

sudo python3 sniffer.py

Generate network traffic in another terminal:

ping google.com

or

curl google.com

The sniffer will capture and display packets generated by these commands.
